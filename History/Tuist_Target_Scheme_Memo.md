# Tuist Target vs Scheme 차이점 및 문제 해결

## 🔍 문제 상황
- Tuist generate 후 `Brake-Release` 스키마가 계속 생성됨
- 커스텀 스키마와 중복으로 생성되어 혼란 발생

## 📋 Target vs Scheme 차이점

### Target (빌드 타겟)
- **정의**: 실제로 빌드할 타겟의 이름
- **위치**: `Projects/App/Project.swift`에서 정의
- **예시**: 
  ```swift
  .app(
      implements: .iOS,
      deploymentTarget: .release,  // ← 여기서 release
      factory: .init(...)
  )
  ```
- **결과**: `Brake-Release`라는 실제 빌드 타겟이 생성됨

### Scheme (Xcode 스키마)
- **정의**: Xcode에서 보이는 스키마의 이름
- **위치**: `DependencyPlugin/ProjectDescriptionHelpers/Scheme.swift`에서 정의
- **예시**:
  ```swift
  let schemeName: String = type == .debug ? "\(name)-\(type.rawValue)" : name
  // debug: "Brake-Debug", release: "Brake"
  ```

## 🚨 문제 원인
Tuist는 **Target의 이름을 기반으로 자동으로 스키마를 생성**함:
1. Target이 `Brake-Release`로 생성됨
2. Tuist가 자동으로 `Brake-Release` 스키마도 함께 생성
3. 커스텀 스키마와 **중복**으로 생성됨

## ✅ 해결 방법

### 방법 1: 자동 스키마 생성 비활성화 (권장)
```swift
// Workspace.swift
let workspace = Workspace(
    name: "Brake",
    projects: ["Projects/**"],
    autogeneratedWorkspaceSchemes: .disabled  // ← 이 옵션 추가
)
```

### 방법 2: Target 이름 명시적 지정
```swift
// Projects/App/Project.swift
.app(
    implements: .iOS,
    deploymentTarget: .release,
    factory: .init(
        name: "Brake",  // ← 명시적으로 이름 지정
        ...
    )
)
```

## 📝 현재 설정 확인
- **Target**: `Brake-Release` (자동 생성)
- **Scheme**: `Brake` (커스텀)
- **문제**: Tuist가 Target 기반으로 자동 스키마 생성

## 🔧 적용 순서
1. `Workspace.swift`에 `autogeneratedWorkspaceSchemes: .disabled` 추가
2. `make clean` 실행
3. `make generate` 실행
4. Xcode에서 스키마 확인

## 📚 참고 사항
- Target과 Scheme은 완전히 다른 개념
- Tuist의 자동 스키마 생성 기능이 기본적으로 활성화되어 있음
- 커스텀 스키마만 사용하려면 자동 생성 기능을 비활성화해야 함
