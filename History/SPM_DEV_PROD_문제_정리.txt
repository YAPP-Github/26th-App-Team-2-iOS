[SPM 번들 DEV/PROD 빌드 오류 해결 과정 정리]

1. 문제 현상
- Tuist 및 Xcode에서 DEV/PROD로 빌드 시, SPM(Swift Package Manager) 패키지 번들(.bundle) 파일이 DEV-iphonesimulator, PROD-iphonesimulator 폴더에 생성되지 않아 'No such file or directory' 에러가 발생함.
- SPM 패키지는 Debug/Release만 지원하고, DEV/PROD는 지원하지 않음.

2. 원인 분석
- SPM은 내부적으로 Debug/Release만 인식하여, DEV/PROD로 빌드해도 번들 파일이 DEV/PROD 폴더에 생성되지 않음.
- Tuist/프로젝트는 DEV/PROD로 빌드 시 해당 폴더에서 SPM 번들을 찾으려 하여 에러 발생.

3. 해결 방법
- xcconfig에서 Debug=DEV, Release=PROD로 매핑하여 내부 코드 분기는 DEV/PROD로 동작하도록 설정.
- Tuist 프로젝트 설정에서 네 가지 configuration(Debug, Release, DEV, PROD)을 모두 등록하여 호환성 확보.
- SPM 번들 파일을 DEV/PROD 빌드 시 Debug/Release 폴더에서 복사하는 빌드 스크립트 추가.

4. 적용한 빌드 스크립트 예시 (App Target의 Project.swift에 추가)

.pre(
  script: """
  if [[ "${CONFIGURATION}" == "DEV" ]]; then
    for bundle in "${BUILT_PRODUCTS_DIR}/../Debug-iphonesimulator/"*.bundle; do
      cp -R "$bundle" "${BUILT_PRODUCTS_DIR}/"
    done
  fi
  if [[ "${CONFIGURATION}" == "PROD" ]]; then
    for bundle in "${BUILT_PRODUCTS_DIR}/../Release-iphonesimulator/"*.bundle; do
      cp -R "$bundle" "${BUILT_PRODUCTS_DIR}/"
    done
  fi
  """,
  name: "Copy SPM Bundles for DEV/PROD"
)

5. 결론 및 주의사항
- SPM 패키지는 DEV/PROD를 지원하지 않으므로, 번들 파일 복사 스크립트로 우회해야 함.
- 경고는 남아있지만, 실제 빌드/실행에는 영향 없음.
- DEV/PROD 환경 분기는 xcconfig와 Swift 코드 내 분기로 처리.

6. 참고
- SPM 공식적으로 Debug/Release만 지원 (https://github.com/apple/swift-package-manager/issues/2620)
- Tuist 공식문서: https://docs.tuist.io/manifests/project/#configurations
