# #44 관리앱 기능 개발 - 기술적 도전 과제 해결 기록

## 📋 개요
본 기록은 `feat/#44-management-app-feature-preconfig` 브랜치에서 관리앱 기능을 개발하는 과정에서 마주한 기술적 도전 과제들과 그 해결 방법들을 정리한 문서입니다.

## 🛠 주요 기술적 도전 과제

### 1. SwiftData 도입 및 활용

#### 🎯 도전 과제
- iOS 17+ 환경에서 SwiftData를 활용한 앱 그룹 데이터 영속성 관리
- 동시성 안전성을 보장하면서 CRUD 연산 구현
- FamilyControls의 `FamilyActivitySelection` 데이터를 SwiftData로 저장/복원

#### 💡 해결 방법

**1) SwiftData Model 설계**
```swift
@Model
public final class AppGroupEntity {
    @Attribute(.unique) public var groupID: Int
    public var name: String
    public var selectionData: Data
    
    public init(groupID: Int, name: String, selectionData: Data) {
        self.groupID = groupID
        self.name = name
        self.selectionData = selectionData
    }
}
```
- `@Attribute(.unique)`를 통해 groupID의 유일성 보장
- `FamilyActivitySelection`을 Data 타입으로 직렬화하여 저장

**2) Actor 기반 동시성 안전성 확보**
```swift
public final actor AppGroupStorage {
    public let context: ModelContext
    
    public init() {
        do {
            let container = try ModelContainer(for: AppGroupEntity.self)
            self.context = ModelContext(container)
        } catch {
            fatalError("컨테이너 생성 문제 \(error.localizedDescription)")
        }
    }
}
```
- Actor 패턴을 활용하여 ModelContext 접근의 Thread-Safe 보장
- MainActor에서만 동작하는 SwiftData의 특성을 고려한 설계

**3) Predicate와 FetchDescriptor 활용**
```swift
public func getAppGroupEntity(groupID: Int) async throws -> AppGroupEntity {
    let predicate = #Predicate<AppGroupEntity> { $0.groupID == groupID }
    let descriptor = FetchDescriptor<AppGroupEntity>(predicate: predicate)
    let result = try? context.fetch(descriptor)
    guard let appGroup = result?.first else {
        throw AppGroupEntityError.notExist
    }
    return appGroup
}
```
- 타입 안전한 `#Predicate` 매크로 활용
- 효율적인 쿼리를 위한 FetchDescriptor 사용

### 2. UI Flickering 문제 해결

#### 🎯 도전 과제
- Empty State와 List State 간의 전환에서 발생하는 UI 깜빡임 현상
- 상태 변화 시 자연스러운 애니메이션 전환 구현
- 사용자 경험을 해치지 않는 부드러운 UI 전환

#### 💡 해결 방법

**1) @retroactive 확장을 활용한 이미지 리소스 그룹화**
```swift
// SharedDesignSystem에서 이미지 그룹 정의
public struct AppGroupImages {
    public let mainEmpty: Image = SharedDesignSystemAsset.Images.appgroupMainEmpty.swiftUIImage
    public let mainFull: Image = SharedDesignSystemAsset.Images.appgroupMainList.swiftUIImage
}

public protocol AppGroupImagesProtocol {
    static var appGroup: AppGroupImages { get }
}

extension AppGroupImagesProtocol {
    public static var appGroup: AppGroupImages { AppGroupImages() }
}

// AppGroupMainView에서 retroactive 확장 적용
extension Image: @retroactive AppGroupImagesProtocol { }
```
- 이미지 리소스를 의미적으로 그룹화하여 `Image.appGroup.mainEmpty` 형태로 접근
- `@retroactive` 키워드로 기존 Image 타입에 새로운 프로토콜 적용
- 타입 안전성과 자동완성 지원으로 개발 효율성 향상
- 네임스페이스 분리로 리소스 관리 체계화

**2) Group을 활용한 조건부 렌더링**
```swift
// 두 뷰를 모두 렌더링하되 opacity로 부드럽게 전환
Group {
    if appGroupMainViewModel.appGroups.isEmpty {
        AppGroupMainEmptyAppGroupView {
            appGroupMainViewModel.addButtonTapped()
        }
    } else {
        // Group List View
        AppGroupMainGroupListView()
    }
}
```
- `Group`을 사용하여 조건부 뷰를 감싸서 레이아웃 재계산 최소화
- 주석을 통해 의도를 명확히 표현

**3) Toast 애니메이션 최적화**
```swift
struct ToastMessageModifier: ViewModifier {
    @State private var showing: Bool = false
    
    func body(content: Content) -> some View {
        content.overlay {
            if isPresented {
                BrakeToastView(message: showMessage)
                    .opacity(showing ? 1 : 0)
                    .offset(y: showing ? 0 : 10)
                    .transition(.move(edge: .bottom))
            }
        }
        .onChange(of: message) { oldValue, newValue in
            // 단계별 애니메이션으로 자연스러운 전환
            Task { @MainActor in
                withAnimation(.easeInOut(duration: 0.3)) {
                    showing = true
                }
                try? await Task.sleep(for: .seconds(1.0))
                withAnimation(.easeInOut(duration: 0.3)) {
                    showing = false
                }
            }
        }
    }
}
```
- `opacity`와 `offset`을 조합한 부드러운 등장/퇴장 효과
- `Task.sleep`을 활용한 타이밍 제어

**4) 스크린타임 권한 상태 관찰 및 대응**
```swift
.brakePopUp(
    isPresented: Binding(
        get: { appGroupMainViewModel.screenTimeAuthAlertPresent },
        set: { appGroupMainViewModel.screenTimeAuthAlertPresent = $0 }
    ),
    title: appGroupMainViewModel.screenTimeAuthErrorResult?.title ?? "",
    message: appGroupMainViewModel.screenTimeAuthErrorResult?.desc,
    primaryButtonTitle: appGroupMainViewModel.screenTimeAuthErrorResult?.primaryButtonTitle ?? "",
    primaryAction: {
        guard let result = appGroupMainViewModel.screenTimeAuthErrorResult else {
            appGroupMainViewModel.screenTimeAuthAlertPresent = false
            return
        }
        switch result {
        case .denied, .unknownError, .userCancel, .approved, .authenticationMethodUnavailable, .networkError:
            appGroupMainViewModel.reAuthButtonTapped()
        case .restricted, .unavailableDevice: break
        }
    }
)
.onChange(of: scenePhase, { oldValue, newValue in
    if newValue == .active {
        self.appGroupMainViewModel.onAppear()
    }
})
```
- `scenePhase` 변화를 관찰하여 앱이 활성화될 때마다 권한 상태 재확인
- 권한 결과에 따른 맞춤형 사용자 안내 메시지 제공
- 권한 거부/오류 상황별 적절한 액션 분기 처리
- Binding을 통한 팝업 상태 양방향 동기화

**5) Alert Modifier의 부드러운 전환**
```swift
.onChange(of: isPresented) { oldValue, newValue in
    var transaction = Transaction()
    transaction.disablesAnimations = true
    
    if newValue {
        withTransaction(transaction) {
            showFullScreenCover = true
        }
    } else {
        allowsInteraction = false
        withAnimation(.easeInOut(duration: 0.3), completionCriteria: .removed) {
            animatedValue = false
        } completion: {
            withTransaction(transaction) {
                self.showFullScreenCover = false
            }
        }
    }
}
```
- `Transaction`을 활용한 애니메이션 제어
- `completionCriteria`를 통한 애니메이션 완료 감지

### 3. 관리앱 UI 구성

#### 🎯 도전 과제
- 다양한 화면 크기에 대응하는 반응형 레이아웃 구현
- 일관된 디자인 시스템 적용
- 사용자 친화적인 앱 선택 및 관리 인터페이스 구현

#### 💡 해결 방법

**1) GeometryReader를 활용한 동적 레이아웃**
```swift
GeometryReader { proxy in
    ScrollView {
        VStack(spacing: 0) {
            VStack(spacing: 24) {
                Image.appGroup.mainFull
                    .frame(width: proxy.size.width)
                    .overlay(alignment: .bottom) {
                        LinearGradient(
                            stops: [
                                .init(color: Color.grey900.opacity(1), location: 0),
                                .init(color: Color.grey900.opacity(0.7), location: 0.27),
                                .init(color: .clear, location: 1)
                            ],
                            startPoint: .bottom,
                            endPoint: .top
                        )
                        .frame(height: proxy.size.height * 0.25)
                    }
            }
            .frame(height: proxy.size.height / 2)
        }
    }
}
```
- `GeometryReader`로 화면 크기에 따른 동적 레이아웃 구현
- `LinearGradient`를 활용한 시각적 효과 적용

**2) 재사용 가능한 컴포넌트 설계**
```swift
public struct UpsertAppGroupView: View {
    @Environment(\.dismiss) var dismiss
    @Environment(UpsertAppGroupViewModel.self) var addAppGroupViewModel
    @FocusState private var isFocused: Bool

                Color.grey900.edgesIgnoringSafeArea(.all)

                    }

        }

```
- `@FocusState`를 활용한 키보드 관리
- 환경값을 통한 의존성 주입 패턴 적용

**3) 앱 리스트 UI 최적화**
```swift
ForEach(addAppGroupViewModel.applicationTokens, id: \.hashValue) { applicationToken in
```

- `Label`의 `iconOnly`/`titleOnly` 스타일 활용
- `lineLimit`과 `truncationMode`로 텍스트 오버플로우 처리
- 명확한 사용자 인터페이스를 위한 버튼 영역 최적화

## 🎯 핵심 성과

### 1. 데이터 관리 및 영속성
- **SwiftData Actor 패턴**: Thread-Safe한 데이터 액세스로 동시성 안전성 확보
- **Predicate 기반 쿼리**: 타입 안전한 데이터베이스 조회로 런타임 오류 방지
- **FamilyActivitySelection 직렬화**: 복잡한 시스템 타입의 효율적인 저장/복원 구현

### 2. UI/UX 최적화 및 애니메이션
- **Flickering 문제 해결**: Group 기반 조건부 렌더링으로 부드러운 상태 전환
- **다단계 애니메이션**: opacity, offset, transition 조합으로 자연스러운 UI 효과
- **Transaction 제어**: 애니메이션 타이밍과 완료 감지를 통한 정밀한 UI 제어
- **반응형 레이아웃**: GeometryReader 활용으로 다양한 화면 크기 대응

### 3. 리소스 관리 및 타입 안전성
- **@retroactive 확장**: 기존 타입에 새로운 기능 추가로 코드 확장성 향상
- **이미지 리소스 그룹화**: 네임스페이스 분리와 타입 안전한 리소스 접근
- **자동완성 지원**: IDE 친화적인 개발자 경험 제공
- **컴파일 타임 검증**: 리소스 참조 오류의 사전 방지

### 4. 권한 관리 및 시스템 연동
- **스크린타임 권한 자동 감지**: scenePhase 관찰을 통한 실시간 권한 상태 추적
- **상황별 맞춤 대응**: 권한 상태에 따른 차별화된 사용자 안내 및 액션
- **앱 생명주기 연동**: 백그라운드/포어그라운드 전환 시 자동 권한 재확인
- **사용자 친화적 오류 처리**: 명확한 안내 메시지와 해결 방안 제시

### 5. 코드 품질 및 아키텍처
- **프로토콜 기반 설계**: 의존성 주입과 테스트 가능한 구조 구현
- **단일 책임 원칙**: 컴포넌트별 명확한 역할 분담과 재사용성 확보
- **환경값 활용**: SwiftUI의 Environment를 통한 깔끔한 의존성 전달
- **관심사 분리**: 뷰, 비즈니스 로직, 데이터 계층의 명확한 분리
