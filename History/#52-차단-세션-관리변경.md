# #52 차단 세션 관리 변경 - 기술적 도전 과제 해결 기록

## 📋 개요
본 기록은 차단 세션 종료 후 사용자 경험 개선을 위한 Shield 화면 관리 로직 변경 과정에서 마주한 기술적 도전 과제들과 그 해결 방법들을 정리한 문서입니다.

## 🎯 개발 배경 및 목표

### 문제 상황
세션이 끝난 이후 "15분 더" 하기 화면이 나타나는 경우를 시환님께 여쭈어보니 다음 경우에만 발생한다고 합니다.
"앱을 사용중인데 세션이 종료되는 경우"

따라서 세션 진행 이후 Flow는 다음과 같습니다:
- 차단 앱을 사용 중(세션)일 때, 세션이 끝남 → 해당 차단한 앱에서 "15분 더" Shield 화면이 나타남
- 그만하기 버튼 누름 → 쿨다운
- 15분 더하기 버튼 누름 → 연장
- 차단 앱에서 홈화면으로 나감 → 쿨다운
- 세션이 끝났는데 차단앱을 이용 안함 → 쿨다운

### 핵심 도전 과제
여기서 감지해야할 것은 크게 두 가지입니다:
1. **차단한 앱을 사용 중인지**
2. **차단 앱의 Shield 화면에서 홈 화면으로 나간 것을 감지할 수 있는지**

이 두 가지 감지에 대한 제약사항이 발생함

## 🛠 주요 기술적 도전 과제

### 1. Shield 화면 상태 관리

#### 🎯 도전 과제
- 세션 종료 시점의 정확한 감지
- Shield 화면 표시 시간 제어
- 사용자 선택에 따른 상태 전환 로직
- 앱 전환 시점의 정확한 감지

#### 💡 해결 방법

**1) 1분 제한 시간 도입**
```
세션 종료 → 1분간 세션 종료 차단창 표시 → 1분이 지나면 쿨다운 창 활성화
```

**2) 외부 앱 접속 상태 관리**
- 세션 종료 → 1분간 세션 종료 차단창 표시 → 1분이 지나면 쿨다운 창 활성화

**3) Brake 앱 접속 상태 관리**
- 외부 상태가 1분간 세션 종료 차단창 표시 상태 → 쿨다운 상태로 전환
- 외부 상태가 세션 종료 차단창에서 연장(스누즈)를 선택함 → 스누즈 휴식 상태로 전환

### 2. 사용자 경험 최적화

#### 🎯 도전 과제
- Shield 화면의 자연스러운 전환
- 사용자 선택의 명확한 피드백
- 시간 제한에 대한 사용자 인지

#### 💡 해결 방법

**1) 내부 상태 기반 쿨다운 관리**
```
내부 상태로 쿨다운 돌아가지 않는 상태로 1분 대기
```

**2) 시간 인지 UI 고려사항**
- 세션 중 화면에서 00분 00초 남은시간으로만 계속 나올 것이라서
- 종료 차단창에 대응하는 메인 앱 화면이 필요
- 내부에서 이 Shield 화면을 띄어줄 수 있는 구조 고려

## 🔄 구현된 플로우

### 세션 종료 후 상태 전환 플로우
```
사용 > 시간 종료 > 종료 차단창 > 쿨다운
```

### 우회 로직 구현
```
세션이 종료됨 → 1분 동안 종료 차단창 → 쿨다운
                      →               (쿨다운 돌아감)
```

### 앱 진입 시점 감지
우리 앱으로 진입한 시점은 알 수 있는데, 이 때 만약 종료 차단창 Shield를 띄어주는 1분간의 상태로 인식하면 바로 쿨 다운으로 이동시키는 방식

## 🎯 기대 효과

### 사용자 경험 개선
- **자연스러운 상태 전환**: 1분 제한 시간을 통한 Shield 화면의 자연스러운 전환
- **명확한 피드백**: 사용자 선택에 따른 즉각적인 상태 변화 피드백
- **시간 인지 향상**: 제한 시간을 통한 사용자의 시간 관리 인식 개선
- **일관된 플로우**: 세션 종료 시 일관된 사용자 경험 제공

### 기술적 안정성
- **상태 관리 일관성**: 외부/내부 앱 상태의 명확한 구분과 일관된 관리
- **앱 전환 감지 정확성**: 앱 진입 시점의 정확한 감지를 통한 상태 전환 안정성
- **예외 상황 처리**: 다양한 사용자 행동 패턴에 대한 안정적인 예외 처리
- **메모리 효율성**: 1분 제한 시간을 통한 불필요한 리소스 사용 방지

### 비즈니스 가치
- **사용자 만족도 향상**: 더 나은 사용자 경험을 통한 앱 만족도 증가
- **기능 완성도**: 차단 세션 관리 기능의 완성도 향상
- **유지보수성**: 체계적인 상태 관리로 인한 코드 유지보수성 개선

## 📝 결과

### 구현 완료 사항
- [ ] 1분 제한 시간 로직 구현
- [ ] 외부/내부 앱 상태 관리 로직 구현
- [ ] Shield 화면 전환 로직 구현
- [ ] 쿨다운 상태 전환 로직 구현

### 향후 개선 사항
- **디자인 시스템과의 통합**: 일관된 UI/UX를 위한 디자인 시스템 적용
- **사용자 피드백 반영**: 실제 사용자 경험을 바탕으로 한 기능 개선
- **성능 최적화**: 상태 전환 시 성능 향상을 위한 최적화 작업
- **외부 Shield 화면과 앱 내부 처리 연동**: 
  - 외부 Shield 화면에서의 사용자 선택이 앱 내부 상태에 자연스럽게 반영
  - 앱 진입 시 외부 상태를 정확히 감지하여 적절한 내부 화면으로 전환
  - Shield 화면과 앱 내부 화면 간의 일관된 상태 관리
  - 사용자가 외부에서 선택한 연장/종료 옵션이 앱 내부에서도 동일하게 적용
